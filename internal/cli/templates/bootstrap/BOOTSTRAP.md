# BOOTSTRAP.md

Use this file for startup context that should be read early in a session.

## Startup Checklist
1. Read `AGENTS.md` for repository rules and non-negotiable constraints.
2. Read `IDENTITY.md`, `SOUL.md`, and `USER.md` for role and collaboration context.
3. Check current issue tracking, active work, and any pending release constraints.
4. Identify the smallest trustworthy path to complete the request.

## Initial Questions To Answer
- What is the user asking for exactly?
- What repository instructions change how the task should be handled?
- What code or docs should be reused before adding anything new?
- What validation is needed before the work is considered done?

- Add short-lived context here when it should shape startup behavior for future sessions.
