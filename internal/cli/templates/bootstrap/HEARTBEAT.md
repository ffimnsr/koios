# HEARTBEAT.md

Use this file for recurring maintenance work that should happen when the main session is idle.

## Routine Checks
- Review open issues, queued follow-ups, and pending cleanup work.
- Look for stale branches of work that need verification or closure.
- Check for lightweight maintenance tasks that can be completed safely.

## Rules
- Prefer low-risk housekeeping over speculative feature work.
- Do not start broad refactors without an explicit request.
- Leave clear notes when a follow-up should be picked up by the next active session.
